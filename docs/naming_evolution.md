# Naming Evolution: Beyond "Agent Framework"

The name "FastAgent" has significant collision with existing projects (e.g., HKUDS/FastAgent) and risks mis-categorizing the project as "just another framework." To reflect its depth as a **Runtime / Kernel / Operating Model**, the following naming paths are proposed.

## 1. OS & Runtime Focus
These names signal that the project is an infrastructure layer, similar to a kernel or a rendering engine.
- **AgentKernel**: Low-level, essential, the core.
- **CognitionOS**: The operating environment for thought.
- **PhaseRuntime**: Emphasizes the explicit execution phases.
- **StateOS**: Focuses on the state-machine nature.
- **DetermiOS**: Highlights the "Deterministic OS" unique selling point.
- **CortexRuntime**: Biological metaphor for a high-level execution layer.
- **SynapseRuntime**: Fast, neural-level execution.
- **ExecutionKernel**: Pure focus on the action layer.

## 2. State & Temporal Focus
These names emphasize the unique CREAM integration and replayability.
- **RetainedAI**: Focuses on the "retained state" and "retained execution graph."
- **TemporalCore**: Highlights the timeline and memory reconstruction.
- **TimelineRuntime**: Direct reference to the observable execution trace.
- **CreamRuntime**: Leveraging the existing CREAM brand as the foundation.
- **StateFlow**: Smooth, observable transitions between states.

## 3. High-Level / Abstract
- **FrameMind**: The "frame" (operating model) for the "mind" (agent).
- **Determina**: A short, punchy brand around determinism.
- **SpatialRuntime**: Linking to the eventual CREAM Spatial OS.

---

## My Recommendations
1. **AgentKernel**: Clean, technical, and immediately explains that it is the "bottom layer" of the agent stack.
2. **CortexRuntime**: Professional, biological, and differentiates from "Frameworks."
3. **DetermiOS**: If the goal is to make "Determinism" the primary brand identity.
